核心内容

GitHub 是让我最开心的地方。我总是腾出时间来做这件事。当我经历艰难的分手时？我在 GitHub 上的开源中迷失了自己。大学期间凌晨 4 点每个人都昏倒了？让我提交一份承诺。在我度蜜月期间，当我妻子还在睡觉时？是的，GitHub。这是我历史上最快乐、最想去的地方。甚至那些烦人的事情！有些人注定会滚动社交媒体。自从这个词出现之前，我就一直在 GitHub 上滚动问题。假期时，我会在 GitHub 上添加我想要研究的不同项目的书签。不仅仅是源代码，还包括 OSS 流程，以及其他维护人员对困难情况的反应。等等，不管你信不信，我喜欢这个。有些人可能会觉得这很恶心，但我的爱好、工作和热情都是一致的，而且在我生命的大部分时间里，它们也必须生活在互联网上的一个地方：GitHub。您是否知道我启动 Vagrant（我的第一个成功的开源项目）很大程度上是因为我希望它能为我在 GitHub 找到一份工作？这不是什么秘密，我已经说过很多次了，在我第一次公开谈论 Vagrant 时，当时我只有 20 岁，我开玩笑说“如果它很好的话，也许 GitHub 会雇用我！” GitHub 是我梦想的工作。我从来没有在那里工作（不是他们的错）。但这是我想去的完美地方。工程师们令人难以置信，产品也令人难以置信，这是我每天生活和呼吸的东西。这 18 年来我仍然这样做，并且一直这样做。足够一个人成长为成年人的时间，全部都在 GitHub 上。最近，我公开批评 GitHub。我对此很刻薄。我一直为此生气。我伤害了人们的感情。我一直在猛烈抨击。因为 GitHub 每天都在让我失望，而且这是针对我个人的。这是非理性的个人行为。我对 GitHub 的爱超过了一个人对某件事物的爱，我对此很生气。对于给从事这项工作的人们带来的伤害，我深表歉意。我已经有这种感觉很长时间了，但在过去的一个月里，我一直在写日记，在 GitHub 中断对我的工作能力产生负面影响的每个日期旁边都画上“X”。几乎每天都有一个 X。在我写这篇文章的那天，我已经有约 2 小时无法进行任何 PR 审查，因为 GitHub Actions 出现故障3。如果这里只是让你每天花几个小时，那么它就不再是一个严肃工作的地方了。这对我来说不再是一个有趣的地方了。

我想去那里，但它不希望我在那里。我想要完成工作，但它不希望我完成工作。我想要发布软件，但它不希望我发布软件。我希望它变得更好，但我也想编码。我无法再使用 GitHub 进行编码了。对不起。

18年后，我该走了。我很想有一天能回来，但这必须基于实际结果和改进，而不是言语和承诺。我将分享有关 Ghostty 项目在未来几个月内迁移到何处的更多详细信息。我们有一个计划，但我仍然在与多个提供商（商业和自由开源软件）进行讨论。我们需要一些时间来删除对 GitHub 的所有依赖项，我们已制定计划尽可能增量地完成此操作。我们计划在 GitHub 上的当前 URL 上保留一个只读镜像。我的个人项目和其他工作目前将保留在 GitHub 上。

Ghostty 是我、我们的维护者和我们的开源社区受影响最大的地方，因此这是此更改的重点。我们将看看之后它会走向何方。脚注 - 这一时间点与 2026 年 4 月 27 日的大规模中断相吻合。几个月来，我们一直在讨论和制定离开 GitHub 的计划，这篇博文是在一周前写的。我们本周才做出最终决定。
