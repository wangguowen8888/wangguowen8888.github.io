WS 首席执行官 Matt Garman 进行了以下采访，内容涉及由 OpenAI 提供支持的 Bedrock Managed Agents；当然，我的问题之一是这如何符合 OpenAI 与 Microsoft 的协议，让 Azure 独家访问 OpenAI 模型。

核心内容

- 周日晚些时候，我通过小道消息得知微软将在周一早上宣布一些事情；我想这会不会是一个先发制人的诉讼！

- 周一，微软和 OpenAI 宣布修改了协议，允许 OpenAI 在其他云提供商（包括 AWS）上提供其产品服务。所以我们到了。我认为微软与 OpenAI 的交易对双方来说都很有意义。以下是微软帖子中新安排的要点： - 微软仍然是 OpenAI 的主要云合作伙伴，OpenAI 产品将首先在 Azure 上发布，除非微软不能并且选择不支持必要的功能。

OpenAI 现在可以通过任何云提供商向客户提供其所有产品。

- 到 2032 年，微软将继续拥有 OpenAI IP 模型和产品的许可。微软的许可现在将是非排他性的。

- 微软将不再向 OpenAI 支付收入分成。

- OpenAI 向微软支付的收入分成将持续到 2030 年，与 OpenAI 的技术进步无关，比例相同，但有总上限。

- 微软作为主要股东继续直接参与 OpenAI 的发展。我认为最重要的一点是最后一点。

Azure 拥有真正的竞争优势，因为它是唯一一家能够提供 OpenAI 模型的超大规模提供商，但这也阻碍了 OpenAI，尤其是当人们清楚许多企业首先关心的是访问其当前选择的云上的模型时；一段时间以来，我一直注意到这对 Anthropic 来说是一个真正的竞争优势。换句话说，Azure 的排他性正在严重损害微软对 OpenAI 的投资，而考虑到 Anthropic 今年的快速增长，微软需要关注他们的投资，即使这会削弱 Azure 的差异化。与此同时，OpenAI 显然将 AWS 视为一个巨大的机会，以至于他们在未来几年放弃了与 Azure 相关的收入（根据之前的观点，这将有助于 Azure 管理层对失去排他性感到更好；他们的损益表在不向 OpenAI 支付收入分成的情况下看起来会好得多）。

OpenAI 还让微软免受 AGI 条款的约束；现在，无论如何，两家公司之间的协议将持续到 2032 年。显而易见的是，OpenAI 的重点将放在 AWS 上，这方面最有力的证据就是本次采访的主题：由 OpenAI 提供支持的 Bedrock Managed Agents。考虑此产品的最简单方法是 AWS 中的 Codex； Codex 之所以能发挥作用，很大程度上是因为它是本地的，这给你带来了很大的复杂性，特别是在安全性方面，而且是免费的。弄清楚如何让代理在整个组织中工作完全是另一回事，该产品的目标是让那些已经在 AWS 中拥有大部分数据的组织更容易访问这些工作流程。为此，在本次采访中，我们讨论了 AWS 如何创建整个云类别、它对初创企业的影响，以及人工智能与之前的范式转变有何相似之处和不同之处。然后我们讨论 Bedrock Managed Agents、它是什么以及它与 Amazon 现有的 AgentCore 产品有何不同。我们还谈到了 Trainium，以及为什么芯片对大多数人工智能用户来说并不重要，以及为什么合作相对于谷歌对全面集成的关注是有意义的。提醒一下，所有 Stratechery 内容（包括采访）都可以播客形式提供；单击此电子邮件顶部的链接将 Stratechery 添加到您的播客播放器。采访内容：OpenAI 首席执行官 Sam Altman 和 AWS 首席执行官 Matt Garman 关于 Bedrock 托管代理的采访 为了清晰起见，本次采访经过轻微编辑。主题：AWS 和初创公司 |基岩托管代理 |本地与云 | AgentCore 与托管代理 | AgentCore 与托管代理培训 |客户需求|构建 AI StackAWS 和初创公司 Matt Garman 和 Sam Altman — 好吧，Matt，欢迎来到 Stratechery — Sam，欢迎回来 [我之前曾于 2025 年 10 月、2025 年 3 月和 2023 年 2 月采访过 Altman]。萨姆·奥尔特曼：谢谢。马特·加曼：谢谢你，谢谢你邀请我。马特，这是你第一次使用 Stratechery。唉，我认为山姆的出现将妨碍通常的“了解你”部分。此外，他不想听我们回忆在凯洛格商学院的时光，但在播客上有一位校友是件好事。

MG：是的，我很高兴来到这里。我下次会回来，我们可以做更深入的探讨。那太好了。您从实习生起就一直在 AWS 工作，现在在人工智能浪潮中负责整个组织。由于缺乏更好的术语，构建人工智能业务的哪些方面与构建原始商品计算业务相同，哪些方面真正不同？

MG：我认为相同的部分是我看到了同样的兴奋和构建者能够做他们以前从未做过的事情，其中​​一件很酷的事情是当我们第一次启动 AWS 时，开发人员突然可以使用只有那些拥有数百万美元来构建数据中心的最大公司才能使用的基础设施。只需一张信用卡和几美元，他们就可以启动应用程序，这确实爆炸了人们在互联网上构建的可能性。我们的想法是，人们可以建造任何他们想要的东西，我们不会预设他们应该做什么，而且世界的创造力是，如果我们能把强大的工具放在他们面前，他们就会建造出有趣和令人惊奇的东西。我认为这对于建筑商能够做的事情来说，即使不是更多，也是同样的变革。当你思考什么是可能的时，你不必去学校学习 10 年编码才能构建应用程序，你不必拥有数百人的庞大团队和数月的时间来构建东西。你可以用小团队来构建东西，你可以快速构建它，你可以快速迭代，人工智能正在世界各个不同方面解锁各种创新。我认为这在很多方面都非常相似，而且看到它为客户群带来的好处真是令人兴奋。不过，当 AWS 出现时，您是唯一的，所以您可以免费获得所有优点和缺点以及一切。在AWS时代，有没有一种感觉，有很多关于商品计算的东西，使其可替代、有弹性、便宜——在人工智能中，特别是在训练中，感觉获胜的抽象更多地是关于这些真正垂直集成的超级集群、真正先进的网络以及软件和硬件之间真正紧密的联系。这对你来说是不是有点惊讶，你现在看到的不是新鲜的，“我们是这里唯一的人，我们有一种特殊的方式来看待大规模计算”，至少在人工智能的最初几年，它可能并不完全一致？

MG：我不知道这对我们来说有什么不同。我认为不同之处在于令人难以置信的快速采用规模，我认为这可能让每个人都感到惊讶。萨姆，如果您不同意，您可以权衡不同的意见，但我认为采用的速度以及人们掌握那里的功能的速度让每个人都感到惊讶。

如果你去了解，当我们开始云计算时，我们花了很长时间来解释为什么书商会为你提供计算能力，这是很多解释来解释什么是云计算。人们忘记了很多艰苦的工作，但早在 2006 年，世界计算就不会朝着这样的方向发展，所以那里有很多艰苦的工作。你是否认为你现在必须做一些解释，因为很多人都在关注训练时代，而你会说，“我们正在考虑推理时代”，这将会是不同的事情，也许你仍然需要再次发挥这些解释力？

MG：确实如此，但人们理解你所谈论的内容的速度完全不同。所以我认为是的，我认为如果你从人们所说的“这看起来确实很酷，而且我有一个可以与之交谈的智能聊天机器人真是太棒了”，转向“我实际上可以在你的企业中工作”，这已经是一种教育，但就技术发展的速度而言，它也相对较快。我保证，我们将很快推出我们想要的产品，但是 Sam - 从初创企业生态系统的角度来看，当你回顾过去时，显然 AWS 具有变革性，完全改变了障碍所在，现在任何人都可以开始。你有种子，你有天使投资者，这在某种程度上消除了截止点的障碍，你不必在 PowerPoint 上获得服务器，你可以构建一个应用程序，然后进入你的 A 轮或任何可能的东西。然而，从您的角度来看，与当今世界相比，有什么不同或相同的地方呢？

SA：我认为大规模初创公司的平台支持有四个伟大时刻：互联网、云、移动、人工智能。第一个让我有点像成年人的东西是云，在 YC [Combinator] 的早期——很难夸大这对初创公司意味着什么。以前，这些初创公司就像租用托管空间，组装服务器，然后把东西放在那里，这是非常复杂的事情，你必须筹集所有这些资金。然后突然间，尽管云是在 YC 刚开始后就出现的，但我猜是在后年。我只是想问——最终，他们真的比你当时意识到的更紧密地联系在一起吗？
